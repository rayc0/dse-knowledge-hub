# DSE情報通信技術（ICT）完全ガイド：日本人受験生のための試験範囲・プログラミング・進学活用

DSE ICTは、コンピュータサイエンス・情報システム・エンジニアリング系への進学を目指す受験生、またはプログラミング経験を活かして高得点を狙いたい受験生に最適な選択科目です。他の理科選択科目と異なり、実践的なスキルが直接得点に結びつく点が特徴です。

## 1. 試験構成と配点

| ペーパー | 内容 | 配点比率 |
|---|---|---|
| Paper 1 | 理論（MCQ + 長問）| 60% |
| Paper 2 | 実践（プログラミング・データベース・ネットワーク設計） | 25% |
| SBA（学校内評価） | ITプロジェクト（設計・開発・評価） | 15% |

Paper 1は理論知識の幅広い確認で、MCQが約30問と長問が含まれます。Paper 2は実際にアルゴリズムを設計したり、疑似コードを書いたり、データベースクエリを作成する実技問題です。SBAはグループまたは個人でのITプロジェクト開発が課題となります。

## 2. 主要トピックと日本の「情報I/II」との比較表

| DSE ICTのトピック | 日本の対応（情報I/情報II） | 深度比較 |
|---|---|---|
| コンピュータシステムの基礎 | 情報Iに含まれる | ほぼ同等 |
| ネットワークとプロトコル（TCP/IP） | 情報IIに含まれる | DSEはより詳細 |
| データ管理・データベース（SQL） | 情報IIに含まれる | DSEはSQLの実践問題あり |
| プログラミング（アルゴリズム） | 情報Iに含まれる | 日本はPython、DSEは疑似コードが中心 |
| サイバーセキュリティ | 情報Iに含まれる | 同等 |
| AIとビッグデータ | 情報IIに一部含まれる | DSEは概念的な出題が多い |
| 社会とIT（倫理・法律） | 情報Iに含まれる | 同等 |
| マルチメディア制作 | 日本の情報Iに一部あり | DSEは実践的プロジェクト形式 |

2022年から開始された日本の「情報I」の内容とDSE ICTは、カバレッジの面でかなり重なっています。情報IIまで学習している生徒は特に有利です。

## 3. プログラミング（疑似コード）の出題傾向

DSE ICTのPaper 2では、プログラミング問題が疑似コード（pseudocode）形式で出題されることが多いです。以下が頻出パターンです。

**頻出アルゴリズム：**
- 配列・リストの線形検索と二分探索
- バブルソート・選択ソートの実装と比較
- 繰り返し構造（for / while）と条件分岐（if-else）
- 再帰関数の基本（階乗・フィボナッチ数列）

**疑似コードの記法例：**
```
INPUT n
SET total ← 0
FOR i ← 1 TO n
    total ← total + i
ENDFOR
OUTPUT total
```

Python経験者は疑似コードに素早く対応できますが、記法の細かい違い（代入演算子 `←`、大文字キーワードなど）をHKEAAの公式記法に合わせる練習が必要です。

## 4. SBAとITプロジェクト作成

SBAでは以下のフェーズで評価されます。

| フェーズ | 内容 | 評価ポイント |
|---|---|---|
| 要件定義 | ユーザーニーズの分析・問題設定 | 明確性・実現可能性 |
| 設計 | フローチャート・データフロー図・UI設計 | 論理的整合性 |
| 実装 | プログラム・データベース・Webの制作 | 動作の正確さ |
| テスト・評価 | バグ修正・改善提案 | 批判的思考力 |

プロジェクトはWebアプリ・モバイルアプリ・データベースシステムなど形式は自由です。採点では「完成度の高さ」より「設計プロセスの論理性と評価の深さ」が重視されます。

## 6. 日本人受験生の強みと弱点

**強み：**
- 日本の「情報I」でプログラミング・ネットワーク・AI基礎を学習済みの生徒は有利
- 数学力を活かしたアルゴリズム設計問題での高得点が期待できる
- SBAのプロジェクト開発でプログラミング経験者は大きなアドバンテージ

**弱み：**
- 英語での技術用語（NIC・DNS・SQL・OSI参照モデルなど）の暗記が必要
- Paper 1の理論問題はHKEAA特有の用語定義があるため、過去問での確認必須
- SBAレポートは英語での技術文書作成が必要

ICT科目はプログラミングや技術系の趣味を持つ日本人受験生にとって、努力量当たりの得点効率が最も高い選択科目の一つです。理工系学部への進学を目指しつつ、得意分野を最大限に活かしたい場合に特に有効です。
