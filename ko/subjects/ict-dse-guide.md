# DSE 정보통신기술（ICT）완전 가이드: 한국인 수험생을 위한 시험 범위·프로그래밍·진학 활용

DSE ICT는 컴퓨터과학·정보시스템·엔지니어링 계열 진학을 목표로 하거나, 프로그래밍 경험을 살려 고득점을 노리는 수험생에게 최적의 선택과목입니다. 다른 이과 선택과목과 달리, 실기 스킬이 직접 점수로 연결되는 것이 특징입니다.

## 1. 시험 구성과 배점

| 페이퍼 | 내용 | 배점 비율 |
|---|---|---|
| Paper 1 | 이론(MCQ + 장문) | 60% |
| Paper 2 | 실기(프로그래밍·데이터베이스·네트워크 설계) | 25% |
| SBA(학교 내 평가) | IT 프로젝트(설계·개발·평가) | 15% |

Paper 1은 이론 지식의 폭넓은 확인으로 MCQ 약 30문항과 장문이 포함됩니다. Paper 2는 실제로 알고리즘을 설계하거나 의사코드를 작성하거나 데이터베이스 쿼리를 만드는 실기 문제입니다. SBA는 그룹 또는 개인 IT 프로젝트 개발이 과제입니다.

## 2. 주요 토픽과 한국 정보 과목과의 비교표

| DSE ICT 토픽 | 한국 정보 과목 대응 | 심도 비교 |
|---|---|---|
| 컴퓨터 시스템 기초 | 정보(2015개정)/정보(2022개정) | 거의 동등 |
| 네트워크와 프로토콜(TCP/IP) | 정보(심화) | DSE가 더 상세 |
| 데이터 관리·데이터베이스(SQL) | 정보(심화) | DSE는 SQL 실기 문제 포함 |
| 프로그래밍(알고리즘) | 정보(2022개정) Python 중심 | 한국은 Python, DSE는 의사코드 중심 |
| 사이버 보안 | 정보(2022개정) | 동등 |
| AI와 빅데이터 | 정보(2022개정)에 일부 포함 | DSE는 개념적 출제 중심 |
| 사회와 IT(윤리·법률) | 정보(2022개정) | 동등 |
| 멀티미디어 제작 | 정보(기초)에 일부 포함 | DSE는 실기 프로젝트 형식 |

2022년 개정 교육과정의 '정보' 과목과 DSE ICT는 커버리지 면에서 상당히 겹칩니다. 정보 심화까지 학습한 학생은 특히 유리합니다.

## 3. 수능 정보 vs DSE ICT: 핵심 차이점

| 비교 항목 | 수능 정보(2022개정) | DSE ICT |
|---|---|---|
| 출제 언어 | 한국어 | 영어 |
| 프로그래밍 언어 | Python | 의사코드(Pseudocode) |
| 실기 평가 | 없음 | SBA 15% |
| 데이터베이스(SQL) | 개론 수준 | 실기 문제로 출제 |
| 네트워크 프로토콜 | 제한적 | 상세 |
| AI/빅데이터 | 포함 | 개념 중심으로 포함 |

## 4. 프로그래밍(의사코드)의 출제 경향

DSE ICT의 Paper 2에서는 프로그래밍 문제가 의사코드(pseudocode) 형식으로 출제되는 경우가 많습니다.

**자주 출제되는 알고리즘:**
- 배열·리스트의 선형 탐색과 이진 탐색
- 버블 정렬·선택 정렬의 구현과 비교
- 반복 구조(for / while)와 조건 분기(if-else)
- 재귀 함수 기초(팩토리얼·피보나치 수열)

**의사코드 표기 예:**
```
INPUT n
SET total ← 0
FOR i ← 1 TO n
    total ← total + i
ENDFOR
OUTPUT total
```

Python 경험자는 의사코드에 빠르게 적응할 수 있지만, 표기법의 세부 차이(대입 연산자 `←`, 대문자 키워드 등)를 HKEAA 공식 표기법에 맞추는 연습이 필요합니다.

## 5. SBA와 IT 프로젝트 작성

| 단계 | 내용 | 평가 포인트 |
|---|---|---|
| 요구 사항 정의 | 사용자 필요 분석·문제 설정 | 명확성·실현 가능성 |
| 설계 | 플로우차트·데이터 흐름도·UI 설계 | 논리적 일관성 |
| 구현 | 프로그램·데이터베이스·웹 제작 | 동작의 정확성 |
| 테스트·평가 | 버그 수정·개선 제안 | 비판적 사고력 |

프로젝트는 웹 앱·모바일 앱·데이터베이스 시스템 등 형식이 자유입니다. 채점에서는 '완성도의 높이'보다 '설계 프로세스의 논리성과 평가의 깊이'가 중시됩니다.

| 대학 | 학부·프로그램 | ICT의 위치 |

ICT는 '필수 과목'으로 지정된 프로그램은 적지만, CS·정보 계열 학부에서는 실질적인 우대 과목으로 취급되며, SBA 프로젝트가 면접·포트폴리오 자료로 활용되기도 합니다.

## 7. 한국인 수험생의 강점과 약점

**강점:**
- 2022 개정 교육과정의 '정보' 과목에서 프로그래밍·네트워크·AI 기초를 학습한 학생은 유리
- 수학력을 활용한 알고리즘 설계 문제에서 고득점 기대 가능
- SBA 프로젝트 개발에서 프로그래밍 경험자는 큰 이점

**약점:**
- 영어로 된 기술 용어(NIC·DNS·SQL·OSI 참조 모델 등) 암기 필요
- Paper 1의 이론 문제는 HKEAA 고유 용어 정의가 있어 기출문제 확인 필수
- SBA 보고서는 영어 기술 문서 작성이 필요

ICT 과목은 프로그래밍이나 기술 계열 취미를 가진 한국인 수험생에게 노력 대비 득점 효율이 가장 높은 선택과목 중 하나입니다. 카카오 스터디 그룹에서 의사코드 문제를 함께 풀거나, 한국어 개인교습(과외)을 통해 영어 기술 논술 표현을 익히는 것이 효과적입니다. 이공계 학부 진학을 목표로 하면서 자신의 강점 분야를 최대한 살리고 싶은 경우 ICT는 특히 유효한 선택입니다.
